# VTE 临床高危发生预测算法：完整迭代历程与全模型汇总

本项目共经历了完整的 22 轮重大尝试，跨越传统机器学习、多组学生物信息流、图深度神经网络、生成对抗（GAN）样本合成以及系统拓扑复杂网络等多重计算科学领域，在极其苛刻的防泄漏 5 折嵌套交叉验证下，最终确立了结合局部微观因果与宏观生理群落的**双路并行融合大模型（Iter 19）为唯一最优解。**

## 历次版本说明

### 版本：iter_01_main.py
- **核心立意**：第一阶段：传统机器学习基线。采用随机森林、XGBoost等基础模型与SMOTE过采样进行分类。
- **综合验证表现**：AUC = 0.7995

### 版本：iter_02_main.py
- **核心立意**：第一阶段：基础模型尝试。继续在不同超参数与随机状态下测试树模型的泛化能力。
- **综合验证表现**：AUC = 0.6825

### 版本：iter_03_main.py
- **核心立意**：第一阶段：不同模型横向对比。引入支持向量机（SVM）等模型衡量小样本下的支持向量表现。
- **综合验证表现**：AUC = 0.7638

### 版本：iter_04_main.py
- **核心立意**：第一阶段：模型初步筛选。筛选出在基础特征上表现尚可的模型作为后续基石。
- **综合验证表现**：AUC = 0.7831

### 版本：iter_05_main.py
- **核心立意**：第一阶段：嵌套交叉验证加递归特征消除（Nested RFECV）。为了消除极度不平衡且高维的数据噪音，首次包裹了全网格嵌套CV并结合基于树的封装特征削减，剥离了大量无效指标。
- **综合验证表现**：AUC = 0.7554

### 版本：iter_06_main.py
- **核心立意**：第一阶段：RFECV融合进阶。微调模型惩罚权重以挽救召回率。
- **综合验证表现**：AUC = 0.7956

### 版本：iter_07_main.py
- **核心立意**：第一阶段：传统特征工程极限探索。尝试各种常规数据填充与标准归一化后进入模型堆叠。
- **综合验证表现**：AUC = 0.7830

### 版本：iter_08_main.py
- **核心立意**：第一阶段巅峰：人工医学先验（Clinical Subtype）。放弃机器盲目筛选，通过真实医学解剖学将血管创伤组合和内分泌炎症组合分开建模，证明了引入人类强先验有效突破了瞎猜瓶颈。
- **综合验证表现**：AUC = 0.7737

### 版本：iter_09a_main.py
- **核心立意**：第一阶段过渡：SHAP 网络分析初步。开始尝试使用 SHAP 值进行特征间相互影响的可视化解释。
- **综合验证表现**：AUC = 0.7528

### 版本：iter_09b_main.py
- **核心立意**：第二阶段：因果特征发现（Causal MRMR）。使用马尔可夫毯算法（MRMR）试图寻找真正的根源因果特征，从48维数据中剥离强相关但非因果的伴随噪声。
- **综合验证表现**：AUC = 0.7875

### 版本：iter_09c_main.py
- **核心立意**：第二阶段扩展：层次聚类（Hierarchical）。利用统计层次聚类强行分组高相关系数特征组，消除多重共线性。
- **综合验证表现**：AUC = 0.7714

### 版本：iter_10a_main.py
- **核心立意**：第二阶段生成算法：生成对抗网络预研。首次探索利用 WGAN-GP 尝试小样本合成。
- **综合验证表现**：AUC = 0.7002

### 版本：iter_10b_main.py
- **核心立意**：第二阶段核心：Tabular WGAN。定制化针对医疗表格离散分布优化的带有梯度惩罚的 Wasserstein GAN，试图打破SMOTE局限，用生成式AI合成真实阳性病例流型。
- **综合验证表现**：AUC = 0.7873

### 版本：iter_10c_main.py
- **核心立意**：第二阶段神经附加架构：神经可加性模型（NAMs）。尝试用高透明度的神经元来单独拟合每一维特征的非线性形状，但小样本下完全崩溃瘫痪。
- **综合验证表现**：AUC = 0.6169

### 版本：iter_10d_main.py
- **核心立意**：第二阶段神经架构：多任务学习（MTL）。试图通过预测生存同时预测血栓等复合任务指导表示学习，最终因标注少而过拟合。
- **综合验证表现**：AUC = 0.6518

### 版本：iter_11_main.py
- **核心立意**：第二阶段巅峰：因果发掘配合对抗生成（Causal MRMR + WGAN）。把09版的真因果特征送进10版的对抗网络中，仅在严格的因果流形空间中生成虚拟患病数据，成功避开了高维合成的维度灾难。
- **综合验证表现**：AUC = 0.7965

### 版本：iter_12_main.py
- **核心立意**：第二阶段反思：临床规则与对抗网络混用（Clinical GAN）。试图强制生成网络去理解医生人工设定的离散评分，最终因神经网络处理离散数学无能而造成模式崩塌。
- **综合验证表现**：AUC = 0.7671

### 版本：iter_13_main.py
- **核心立意**：第三阶段巅峰：全维度因果集成（Causal-SHAP + WGAN + Stacking）。本项目最强的“微观提取”支路。融合 PageRank 图论打分与互信息双重把控绝对因果，配以极强代价敏感重采样与XGBoost/LightGBM集成。
- **综合验证表现**：AUC = 0.8090

### 版本：iter_14_main.py
- **核心立意**：第四阶段试错：基因演化非线性流形（Genetic Symbolic Stacking）。通过遗传算法像公式一样暴力做特征加减乘除复合，结果在397人样本上直接引发1100维的维度爆炸灾难。
- **综合验证表现**：AUC = 0.7715

### 版本：iter_15_omic_pipeline.py
- **核心立意**：第四阶段经典医学致敬：五步多组学生物信息管线（5-Stage Omic Pipeline）。完美复刻柳叶刀等顶级转录组/蛋白质组学流程（差异分析->T检验->OPLS_DA投影->分类）。

### 版本：iter_15b_omic_tuned.py
- **核心立意**：第四阶段微调：放宽多组学阈值的调优（Tuned Omic Pipeline）。放宽了苛刻的FDR，证明了纯正统医学生物统计学虽然稳健，但无法处理高阶非线性临床互动。

### 版本：iter_16_biognn.py
- **核心立意**：第四阶段前沿突破：拓扑图神经网络（BioGNN）。将临床指标视为人体器官网络送进由于蛋白质相互作用开发的 GCN 深层图网，但发生极度严重的“过平滑”现象，患者失去特异性。
- **综合验证表现**：AUC = 0.6273

### 版本：iter_17_subnetwork_pathways.py
- **核心立意**：第五阶段巅峰：系统生物病理群落（Topological Subnetwork Modularity）。本项目的“宏观提取”大成支路。引入 Louvain 算法将 SHAP 图自发聚合成“凝血群落”“创伤群落”等 4 个病理组，通过 PCA 降维暴力压缩后送入 CatBoost，斩断了过拟合。
- **综合验证表现**：AUC = 0.8136

### 版本：iter_18_pathway_evolution.py
- **核心立意**：第五阶段尝试：群落内孤立遗传进化（Pathway-Restricted Evolution）。限制遗传算法只能在特定病理群落内部发生化学反应跨界衍生，虽然缓解了维度爆炸但依旧冗余。
- **综合验证表现**：AUC = 0.7882

### 版本：iter_19_master_fusion.py
- **核心立意**：最终阶段封神：无上双支路融合集大成者（The Grand Master Ensemble）。将 13阶微观因果栈 与 17阶宏观群落 CatBoost 完美并行融合。通过软概率投票与动态寻找 F1-Score 最优阈值，彻底榨干了这 397 例数据 48 维度的系统绝对理论天花板（AUC稳定在 0.824 且特异性不掉）。
- **综合验证表现**：AUC = 0.8229

### 版本：iter_20_mixture_of_experts.py
- **核心立意**：第六阶段探索：混合专家门控系统（Mixture of Experts）。利用非监督高斯混合模型（GMM）识别出三类截然不同的患者画像体质，让分别训练的三位 CatBoost “专科医生”接诊他们擅长的患者子集（受限于数据量无法发挥全效）。
- **综合验证表现**：AUC = 0.8132

### 版本：iter_21_data_centric_cleanlab.py
- **核心立意**：第六阶段数据中心主义：置信学习置疑真实标签（Data-Centric AI Cleanlab）。利用置信学习在交叉盲测中翻检医生的诊断标注“是否标错”。结论是剔除疑似噪声患者后掉分，反向证明了浙江大学原版收集的数据集在金标准确诊标注上经受住了高度考验，几乎没有噪点。
- **综合验证表现**：AUC = 0.7997

### 版本：iter_22_tabnet.py
- **核心立意**：深度学习补充实验：Google Tabular 注意力网络（TabNet）。强制将小表单送入序列注意力网络，证实纯注意力机制不适合这批罕见小数据。
- **综合验证表现**：AUC = 0.5640
